## 🎯 Sobre o Projeto

O projeto **Simulação de Atendimento no Node.js REPL** foi desenvolvido como uma atividade acadêmica individual, com o objetivo de praticar comandos básicos de JavaScript no terminal do Node.js.

A proposta consiste em simular o atendimento de um visitante em uma feira de tecnologia escolar, aplicando estruturas fundamentais da programação.

Projeto desenvolvido individualmente por **Breno José de Oliveira**.

---

## ⚙️ Objetivo da Atividade

A atividade teve como finalidade simular um sistema de atendimento utilizando o Node.js.

Foram executadas as seguintes etapas:

- abrir o Node.js no terminal  
- exibir mensagem inicial  
- mostrar a data atual  
- declarar variáveis  
- exibir dados do visitante  
- validar informações com if  
- exibir erro e aviso  
- mostrar dados em tabela  
- usar while, for e forEach  
- criar função de saudação  
- gerar arquivo saida.txt  

---

## 👤 Dados do Visitante

- Nome: Lucas  
- Idade: 17 anos  
- Quantidade de ingressos: 3  
- Valor do ingresso: R$ 20,00

Após isso, foram executados comandos JavaScript diretamente no terminal do Node.js.

## 🧠 Explicação da Resolução

Primeiramente foi exibida uma mensagem indicando o início do sistema de atendimento.

Depois disso, foi mostrada a data atual utilizando o objeto Date, formatada para o padrão brasileiro.

Em seguida, foram criadas variáveis contendo os dados do visitante. O valor total da compra foi calculado automaticamente.

Os dados foram exibidos no terminal de forma organizada.

A idade foi validada com uma estrutura condicional. Caso fosse inválida, seria exibido um erro. Como a idade é 17, foi considerada válida.

A quantidade de ingressos também foi validada. Como o valor é menor que 5, não houve aviso.

Foi utilizada uma tabela para exibir o resumo da compra.

Depois disso, foi utilizado um laço while para simular a emissão de senhas.

Um laço for foi usado para listar os ingressos.

O método forEach foi aplicado para mostrar os setores do evento.

Foi criada uma função de saudação para o visitante.

Por fim, foi gerado um arquivo chamado saida.txt contendo o resumo do atendimento.
